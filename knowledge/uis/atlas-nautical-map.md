# Atlas 航海星图

## 定位

Atlas 航海星图是 Scroll Vessel 分类导航的核心 UI 隐喻。它把分类体系表达为一张可探索的知识航图，用于帮助读者理解内容结构、发现主题区域，并进入具体分类页。

## 适用范围

- Atlas 分类地图页。
- 首页通向 Atlas 的核心入口预览。
- 分类页头部中用于表达当前位置的轻量航图元素。

不适用于文章正文阅读区。文章页应继续以可读性、SEO 和可访问性为优先，遵循 [`scroll-vessel-experience.md`](scroll-vessel-experience.md)。

## 核心隐喻

- 一级分类表达为海域、星区或主要航线区域。
- 子分类表达为岛屿、灯塔、航标、港口或节点。
- 分类之间的父子关系表达为航线、虚线、等高线或区域边界。
- 文章数量表达为节点尺寸、密度、旁注或数字徽标。
- 最近更新或重点分类可以通过高亮航线、节点强调或详情面板表达。

## 信息结构

Atlas 页面至少包含：

1. 页面标题和一句定位说明，明确这是分类地图而不是普通文章列表。
2. 航海星图区，展示分类层级、文章数量和可点击入口。
3. 节点详情区，用于展示当前 hover、focus 或选中分类的简介、文章数、最近更新和进入分类页动作。
4. 列表替代视图，用于移动端、键盘用户、屏幕阅读器和减少复杂视觉负担的场景。

## 视觉语言

- 整体气质是技术出版物中的航线图，而不是游戏地图或装饰插画。
- 背景可使用纸感、浅色网格、坐标线、星图线或航海图纹理，但不能影响文字对比度。
- 线条应克制，服务于层级和路径关系。
- 色彩以高可读的中性色为主，使用少量品牌强调色标记当前节点、主航线或重点区域。
- 字体层级应突出 Atlas 的标题感，同时保证分类名称和数量信息清晰可读。

## 交互规则

- 节点必须可点击并进入对应分类页。
- hover、focus 和 selected 状态应清晰区分。
- hover 或 focus 节点时，详情区同步展示分类摘要和进入动作。
- 不能只依赖 hover；触屏和键盘路径必须可完成同等任务。
- 动效用于表达路径显现、节点聚焦或区域切换，时长应保持克制，并支持 `prefers-reduced-motion`。
- 返回、深链接和浏览器前进后退应保持可预测。

## 可访问性要求

- 所有分类节点必须能通过键盘访问。
- Tab 顺序应匹配视觉阅读顺序或列表替代视图顺序。
- 当前节点状态不能只靠颜色表达。
- 图形化航图应有语义化的列表或导航结构作为等价入口。
- 页面标题层级必须连续，不因视觉大小跳过 heading 级别。
- 文字和节点标签必须满足对比度要求。

## 响应式要求

- 桌面端可以优先展示完整航图和详情面板。
- 平板端可以降低节点密度，保持核心分类和详情面板。
- 移动端应优先保证可用性，可以将航图简化为横向分区、纵向航线或直接优先展示列表替代视图。
- 不允许为了保留地图效果造成水平滚动或不可点击的小节点。

## 与其他 UI 知识的关系

- 本文件定义 Atlas 航海星图隐喻和交互原则。
- [`scroll-vessel-experience.md`](scroll-vessel-experience.md) 定义 Scroll Vessel 整体体验标准。
- 具体 story 的 `ui.md` 只应引用本文件，并在 story-specific UI 产物中说明本次采用哪些部分，不复制本文件正文。
