# 逐步细化

把粗糙意图通过阶段化过程转成可实现工作：

```text
references/wips/
→ products/*/mvp.md
→ products/*/user-journey-*/user-journey.md
→ products/*/user-journey-*/story-*/story.md
→ story ui.md + tests.md
→ story plans/plan.md
→ 实现
→ plans/qa.md、review.md、progress.md
```

除非用户明确要求一次性修改，否则产品工作不应从草案直接跳到实现。应先经过 MVP、用户旅程、故事、UI/测试和计划。
