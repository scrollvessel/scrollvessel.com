# 内容源校验链路

## 定位

本链路从测试视角定义 Markdown 内容源进入内容索引前的验收路径。它不是某个 story 的测试用例，而是可被多个 story 引用的 QA 链路。

## 链路总览

```text
内容目录存在
→ Markdown 文件被发现
→ 路径映射稳定 URL
→ Front Matter 被解析
→ 必填字段校验
→ 字段类型校验
→ 语义规则校验
→ 内容事实归一
→ 分类上下文生成
→ 错误报告可定位
→ 内容源记录进入内容索引
```

## 01. 内容目录存在

### 验收目标

内容扫描入口能定位约定的内容目录，并能处理空目录或缺失目录。

### 通过标准

- 内容目录存在时，扫描流程能读取目录。
- 空目录返回可理解的空内容状态。
- 缺失目录产生可定位错误或明确的空状态策略。

### 失败标准

- 因空目录产生不可定位异常。
- 缺失目录时错误不说明路径。

### 边界情况

- 目录为空。
- 目录不存在。
- 目录存在但没有 Markdown 文件。

### 推荐证据

- 单元测试覆盖空目录和缺失目录。
- 错误信息包含目录路径。

## 02. Markdown 文件被发现

### 验收目标

扫描流程能发现内容目录下的 Markdown 文件，并忽略非内容文件。

### 通过标准

- `.md` 文件被纳入扫描候选。
- 非 Markdown 文件不被误判为文章。
- 子目录中的 Markdown 文件可被发现。

### 失败标准

- 漏扫子目录文章。
- 把图片、配置或其他资源误识别为文章。

### 边界情况

- 深层目录。
- 文件名包含空格或非 ASCII 字符。
- 同目录同时存在 Markdown 和资源文件。

### 推荐证据

- 单元测试断言发现文件列表。
- 测试数据包含嵌套目录和资源文件。

## 03. 路径映射稳定 URL

### 验收目标

文件路径能稳定映射为内容 URL，URL 稳定性来自内容路径稳定性。

### 通过标准

- 相同路径在多次扫描中生成相同 URL。
- URL 结构与 [`../../architectures/static-markdown-content.md`](../../architectures/static-markdown-content.md) 一致。
- 路径冲突能被发现。

### 失败标准

- 同一文件生成不稳定 URL。
- 两个文件生成相同 URL 却未报错。

### 边界情况

- `index.md` 与同名目录。
- 大小写差异。
- 重名 slug。

### 推荐证据

- 单元测试断言路径到 URL 的映射。
- 冲突测试断言错误包含冲突文件路径。

## 04. Front Matter 被解析

### 验收目标

每篇 Markdown 的 Front Matter 能被解析为结构化内容事实。

### 通过标准

- YAML 语法合法时可解析。
- YAML 语法错误时校验失败并定位文件。
- 正文与 Front Matter 边界清晰。

### 失败标准

- YAML 错误被静默吞掉。
- 正文内容被误解析为 Front Matter。

### 边界情况

- 缺少 Front Matter 分隔符。
- YAML 语法错误。
- Front Matter 为空。

### 推荐证据

- 单元测试覆盖合法、缺失、空和语法错误 Front Matter。
- 错误消息包含文件路径。

## 05. 必填字段校验

### 验收目标

所有文章必须具备 [`../../domains/article-frontmatter.md`](../../domains/article-frontmatter.md) 定义的必填字段。

### 通过标准

- 必填字段存在且非空。
- 缺少任一必填字段时校验失败。
- 错误指出文件、字段和原因。

### 失败标准

- 缺少必填字段仍进入内容索引。
- 错误只显示通用失败，无法定位字段。

### 边界情况

- 字段缺失。
- 字段为空字符串。
- 字段名拼写错误。
- 字段存在但值为 `null`。

### 推荐证据

- 单元测试覆盖每个必填字段缺失。
- 错误断言包含字段名。

## 06. 字段类型校验

### 验收目标

Front Matter 字段类型符合 [`../../domains/article-frontmatter.md`](../../domains/article-frontmatter.md)。

### 通过标准

- 字符串、日期、布尔值、数组和外链结构类型正确。
- 类型错误时校验失败或按权威规则降级。

### 失败标准

- 类型错误字段被当作合法事实使用。
- 日期格式错误但未被发现。

### 边界情况

- 日期格式错误。
- `tags` 不是数组。
- `featured` 或 `draft` 不是布尔值。
- `externalLinks` 不是数组。

### 推荐证据

- 单元测试覆盖代表性类型错误。
- 错误断言包含字段路径。

## 07. 语义规则校验

### 验收目标

字段不仅类型正确，还应符合内容语义规则。

### 通过标准

- `createdAt`、`updatedAt`、`draft`、`cover`、`externalLinks` 等字段按领域规则解释。
- 结构化数据由构建层根据内容事实生成，Front Matter 不承载派生结果。

### 失败标准

- 把派生 SEO 或 JSON-LD 结果写回 Front Matter。
- 字段含义和 [`../../domains/article-frontmatter.md`](../../domains/article-frontmatter.md) 冲突。

### 边界情况

- `updatedAt` 早于 `createdAt`。
- `cover` 路径不可用。
- `externalLinks` 字段部分缺失。

### 推荐证据

- 单元测试覆盖语义异常。
- 对可降级字段断言保留可用字段。

## 08. 内容事实归一

### 验收目标

扫描结果能把 Front Matter、路径和文件上下文归一为稳定内容事实。

### 通过标准

- `draft`、demo 标识、`tags`、`externalLinks` 和资源上下文被结构化保留。
- 草稿发布规则符合 [`../content-site-quality.md`](../content-site-quality.md)。
- demo 内容可被识别，供页面明确标注。

### 失败标准

- 草稿被当作正式生产内容发布。
- demo 信息丢失，导致页面无法标注。

### 边界情况

- 同时存在 `draft` 和 demo 标识。
- 没有 tags。
- 没有 externalLinks。

### 推荐证据

- 单元测试断言内容事实结构。
- 生产构建路径覆盖草稿排除规则。

## 09. 分类上下文生成

### 验收目标

内容源记录应携带分类上下文，供首页、Atlas、分类页、文章页、SEO 和订阅产物复用。

### 通过标准

- 分类上下文来自目录结构。
- 目录可同时包含子目录和 Markdown 文件。
- 分类层级和文章数量可由同一内容索引推导。

### 失败标准

- 分类上下文丢失。
- 目录混合结构导致扫描失败。
- 不同页面各自生成不一致分类数据。

### 边界情况

- 空分类。
- 只有子分类没有文章。
- 同层级多篇文章。

### 推荐证据

- 单元测试断言分类路径。
- 索引测试断言分类与文章关联。

## 10. 错误报告可定位

### 验收目标

内容错误应让作者知道哪个文件、哪个字段、什么原因、如何修复。

### 通过标准

- 开发环境错误信息包含文件、字段、原因和修复方向。
- 生产构建可以简洁失败，但必须指出阻塞原因。
- 多个错误可以被区分展示。

### 失败标准

- 只输出通用错误。
- 多文件错误混在一起无法定位。

### 边界情况

- 单文件多错误。
- 多文件多错误。
- YAML 解析错误和字段校验错误同时存在。

### 推荐证据

- 错误报告单元测试或快照。
- 构建失败输出包含关键路径和字段。

## 11. 内容源记录进入内容索引

### 验收目标

通过校验的内容源记录能进入统一内容索引，供后续链路使用。

### 通过标准

- 合法文章进入内容索引。
- 不合法文章阻塞索引或构建。
- 内容索引作为后续页面、SEO、RSS 和分类数据的同一来源。

### 失败标准

- 不合法文章进入索引。
- 合法文章缺失。
- 后续产物绕过内容索引另建数据源。

### 边界情况

- 部分文章合法、部分文章不合法。
- 草稿在开发和生产中的索引策略不同。

### 推荐证据

- 索引结构断言。
- 构建接入测试。
